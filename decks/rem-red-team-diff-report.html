Behavior And Contracts

  • add --source X --source-id Y is idempotent for identical bodies and rejects implicit updates. Opaque IDs such as null, list-like values, quotes, and surrounding spaces round-trip exactly.
  • review never writes. Exact matches return NoOp; explicit targets return Update; same facts return Append; changed exclusive relations return Supersede; compatible or ambiguous candidates return Add.
  • supersede writes one replacement, marks the old memory superseded, records supersedes, and excludes the old fact from current semantic queries. Explicit metadata overrides prevent an unchanged-body NoOp.
  • Every mutating command, dry-run, and rebuild now acquires one vault lock. A competing process fails before writing; stale locks are reported by doctor.
  • Reindex diagnostics now reject duplicate source identities and malformed/duplicate frontmatter while preserving the previous SQLite cache on failure.
  • Git hooks that leave post-commit vault changes trigger rollback of HEAD, Markdown, SQLite, and staging instead of returning success with a dirty tree.
