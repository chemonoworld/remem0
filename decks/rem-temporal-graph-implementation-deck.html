north star

핵심 설계 결정

Canonical

Markdown + Git 유지

메모리의 durable source는 계속 Markdown이고, commit/rollback의 경계는 기존 Git transaction core다.

Derived Cache

SQLite에 temporal semantic index 추가

Graph DB를 붙이지 않고 rebuild 가능한 SQLite cache에 entity, fact, episode, ontology tables를 만든다.

user contract

새로운 Markdown directive와 CLI

@fact User | PREFERS | Adidas | valid_from=2025-01-10 | valid_to=2025-04-02 @fact User | PREFERS | Puma | valid_from=2025-04-02
rem facts --entity User rem facts --relation works-at rem facts --at 2025-02-01 rem facts --all rem facts --source

기본 조회는 current facts만 보여주고, `--at`은 historical view, `--source`는 provenance를 함께 보여준다.

ontology

작고 통제된 ontology로 시작

PREFERS

선호. 이전 current fact를 닫을 수 있는 relation.

USES

도구/서비스 사용. 여러 current fact 허용.

WORKS_AT

소속/직장. 보통 하나의 current가 자연스러움.

MENTIONS

약한 언급. invalidation rule 없음.

전체 relation set: `PREFERS`, `DISLIKES`, `USES`, `WORKS_AT`, `HAS_PROJECT`, `PART_OF`, `SUPERSEDES`, `MENTIONS`.

schema

SQLite semantic cache schema

semantic_entitiescanonical_name, kind, summary, timestamps
semantic_episodessource memory, text hash, excerpt, source path
semantic_factssubject, relation, object, validity window, confidence
semantic_fact_sourcesfact to episode provenance link
semantic_ontology_relationsrelation whitelist and policy metadata
transaction integration

rebuild/commit 흐름 안에 semantic validation을 넣었다

1. Markdown read기존 memory parser로 canonical files 수집
2. FTS index기존 memories / memories_fts insert
3. Semantic extract`@fact` directive를 entity/fact/episode로 변환
4. Diagnosticsunsupported relation, bad time, duplicate fact id는 diagnostic
5. Transactiondiagnostic 발생 시 strict reindex 실패로 rollback
red team fixes

레드팀 루프에서 실제로 잡은 버그

Temporal query correctness

  • 미래 `valid_from` fact가 current에 섞이던 문제 차단.
  • 미래 `valid_to`가 있는 bounded fact가 current에서 빠지던 문제 수정.
  • unix seconds를 TEXT가 아니라 numeric comparison으로 비교.
  • 역전된 `valid_from` / `valid_to`, mixed time format, non-padded date 검증.

Directive parser safety

  • Markdown fenced code와 indented code 안의 `@fact` 무시.
  • 빈 segment, duplicate option, invalid confidence 차단.
  • `object_kind=value`와 relation normalization 테스트.
  • invalid `rem facts --at` 입력을 user error로 처리.

review focus

리뷰어가 집중할 부분

Semantic model

  • Relation whitelist가 MVP로 충분한지.
  • `@fact` directive를 Markdown body에 두는 UX가 괜찮은지.
  • `object_kind=value` 처리와 entity id normalization이 기대와 맞는지.

Temporal semantics

  • `valid_to`를 exclusive upper bound로 둔 결정.
  • 허용 time format: unix seconds, `YYYY-MM-DD`, `YYYY-MM-DDTHH:MM:SSZ`.
  • current query가 runtime clock을 쓰는 점.
risks and follow-ups

아직 의도적으로 남긴 것

Not implemented

  • LLM 기반 자동 fact extraction.
  • semantic conflict review UI.
  • entity merge review and canonical merge decisions.
  • external graph database integration.

Current assumption

  • Semantic cache는 derived data이고 Git에 commit하지 않는다.
  • 잘못된 semantic directive는 reindex diagnostic으로 보고 transaction rollback한다.
  • Ontology는 작게 시작하고, relation explosion은 whitelist로 막는다.
